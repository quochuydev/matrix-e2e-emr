# Matrix Patient Records

E2EE patient records on top of Matrix. Each patient is a private encrypted
room; the profile is the root event of an `m.thread`, and revisions are
replies to that root. Messages live in the same room's main timeline.

## Architecture

The app has five independent flows. The diagrams below cover each one in
isolation; together they make up the whole.

### 1. Layers at a glance

```mermaid
flowchart LR
    UI["React UI<br/>(client components)"] --> Provider["MatrixProvider<br/>Context + recovery-key gate"]
    Provider --> Lib["src/lib/matrix<br/>thin wrappers"]
    Lib --> SDK["matrix-js-sdk<br/>(+ rust crypto)"]
    SDK <--> IDB[("IndexedDB")]
    SDK <--> Synapse[("Synapse homeserver")]
```

### 2. Sign-in & unlock

```mermaid
sequenceDiagram
    participant U as User
    participant F as Sign-in form
    participant P as MatrixProvider
    participant C as client.ts
    participant S as secret-storage.ts
    participant H as Synapse

    U->>F: username + password + recovery key
    F->>P: signIn()
    P->>C: loginWithPassword()
    C->>H: POST /login (m.login.password)
    H-->>C: access_token, user_id, device_id
    C-->>P: StoredSession (saved to localStorage)
    P->>C: createMatrixClient()
    C->>C: initRustCrypto() + startClient()
    C->>H: /sync (wait for PREPARED)
    P->>S: unlockWithSecurityKey(recoveryKey)
    S->>H: fetch m.secret_storage.key.<id>
    S->>S: decodeRecoveryKey + checkKey
    S->>S: loadSessionBackupPrivateKeyFromSecretStorage
    S->>H: restoreKeyBackup
    P->>P: keyUnlockedThisSession = true → ready
```

### 3. Patient CRUD (encrypted room per patient)

```mermaid
flowchart TB
    UI["Create / Edit dialog"] -->|createPatient| Patients["patients.ts"]
    UI -->|updatePatient| Patients
    UI -->|deletePatient| Patients

    Patients -->|createRoom + encryption state| SDK
    Patients -->|setRoomTag PATIENT_TAG| SDK
    Patients -->|sendEvent PATIENT_RECORD_EVENT_TYPE| SDK
    Patients -->|sendStateEvent PROFILE_THREAD_STATE_TYPE| SDK
    Patients -->|getUserDeviceInfo for invitees| Crypto["crypto-api"]
    Patients -->|leave + forget| SDK

    SDK["matrix-js-sdk"] <--> Synapse[("Synapse")]
    Crypto <--> Synapse
```

Profile is the root of an `m.thread`; edits are replies that point at it
via `m.relates_to`. Latest reply wins when rendering.

### 4. Messaging & live updates

```mermaid
sequenceDiagram
    participant D as patient-detail.tsx
    participant Pt as patients.ts
    participant C as MatrixClient
    participant H as Synapse

    Note over D,Pt: Mount
    D->>Pt: subscribeRooms(client, refresh)
    Pt->>C: on ClientEvent.Room
    Pt->>C: on RoomEvent.Timeline / Tags / Name
    Pt->>C: on MatrixEventEvent.Decrypted

    Note over D,H: Initial render
    D->>Pt: listMessages / listPatientHistory

    Pt-->>D: filtered events

    Note over D,H: User sends a message
    D->>Pt: sendMessage(text)
    Pt->>C: sendEvent(EventType.RoomMessage)
    C->>H: encrypted PUT
    H-->>C: /sync delivers echo + remote events
    C-->>Pt: emits RoomEvent.Timeline / Decrypted
    Pt-->>D: refresh() re-reads timeline
```

### 5. Recovery key & key backup

```mermaid
sequenceDiagram
    participant U as User
    participant Bar as Status bar
    participant P as MatrixProvider
    participant S as secret-storage.ts
    participant K as crypto-api
    participant H as Synapse

    rect rgb(245,245,245)
    Note over U,H: First-time setup (generate)
    U->>Bar: Generate recovery key
    Bar->>S: generateRecoveryKey()
    S->>K: createRecoveryKeyFromPassphrase
    S->>K: bootstrapCrossSigning
    S->>K: bootstrapSecretStorage(setupNewKeyBackup)
    K->>H: write SSSS + new backup version
    S-->>U: display key once
    end

    rect rgb(245,245,245)
    Note over U,H: Background upload
    K-->>P: CryptoEvent.KeyBackupSessionsRemaining
    P->>P: pendingBackup → block sign-out / warn on unload
    end

    rect rgb(245,245,245)
    Note over U,H: Reset (lost backup)
    U->>Bar: Reset backup + key
    Bar->>P: resetBackup(key)
    P->>S: cacheSecurityKey
    P->>K: resetKeyBackup
    P->>K: loadSessionBackupPrivateKeyFromSecretStorage
    P->>P: decryptAllEvents per room
    end
```

### 6. Invites

```mermaid
sequenceDiagram
    participant U as User (inviter)
    participant Pt as patients.ts
    participant C as MatrixClient
    participant H as Synapse
    participant U2 as User (invitee)
    participant P2 as MatrixProvider (invitee)
    participant B as Invites banner

    Note over U,H: Outgoing invite at room creation
    U->>Pt: createPatient(input, inviteUserIds)
    Pt->>C: createRoom with invite list
    Pt->>C: getUserDeviceInfo(targets) to pre-share megolm
    C->>H: room state + invite events

    Note over H,B: Incoming invite (invitee)
    H-->>C: sync delivers invite-state
    C-->>P2: emits RoomEvent.MyMembership
    P2->>Pt: listPendingInvites
    Pt-->>B: PendingInvite list

    Note over U2,H: Accept or decline
    U2->>B: Accept
    B->>Pt: acceptPatientInvite
    Pt->>C: joinRoom + setRoomTag PATIENT_TAG
    U2->>B: Decline
    B->>Pt: declinePatientInvite
    Pt->>C: leave
```

### 7. Sign-out & wipe

```mermaid
flowchart LR
    U["User clicks sign-out"] --> P["MatrixProvider"]
    P -->|guard pendingBackup > 0| Warn["Block + toast"]
    P -->|teardownClient| Steps
    Steps["1. logout(true)<br/>2. stopClient()<br/>3. clearStores()<br/>4. wipeLocalMatrixData()"] --> Browser

    Browser["localStorage + IndexedDB"]
    Steps -.delete prefixes.-> Browser
```

Wiped prefixes: IndexedDB `matrix-app:`, `matrix-app-crypto:`,
`matrix-js-sdk:`, `@matrix-org/`; localStorage `mx_`, `matrix-app.`,
`matrix-js-sdk`.

## Development

```bash
docker compose up -d   # local Synapse homeserver
pnpm install
pnpm dev
```

Sign in with a Matrix account on the local homeserver. The first session
must generate a recovery key (shown once); subsequent sessions need that
key to unlock — per the rule in `AGENTS.md`, no feature is usable until
the recovery key is entered.
